\noindent\textbf{Four pipeline replay variants.}
Each session page below arranges the same four plots in a $2\times2$ grid (top: binary path; bottom: multiclass path; left: SVM; right: LSTM).
\begin{center}
\footnotesize
\setlength{\tabcolsep}{0.6em}

\begin{tabular}{@{}>{\raggedright\arraybackslash}p{0.47\linewidth}>{\raggedright\arraybackslash}p{0.47\linewidth}@{}}
\multicolumn{2}{@{}c@{}}{\textit{Top row --- binary locomotion (assist vs.\ not), then FSM}} \\[0.35em]
\textbf{Binary SVM + FSM} &
\textbf{Binary LSTM + FSM} \\
Sliding-window IMU features feed a trained RBF SVM; the predicted binary class drives \texttt{RealtimeFsm} pseudo-real-time assist gating and the replay time series (vs.\ ground truth and IMU). &
Same windowing and binary labels as the SVM path, but a trained LSTM outputs the binary class each step; FSM loop and plot layout match the SVM replay for direct comparison. \\[0.65em]
\multicolumn{2}{@{}c@{}}{\textit{Bottom row --- twelve activity classes, mapped to locomotion FSM}} \\[0.35em]
\textbf{Multiclass ECOC SVM + FSM} &
\textbf{Multiclass LSTM + FSM} \\
ECOC SVM predicts twelve HuGaDB activity classes per window; class $\rightarrow$ locomotion mapping feeds the multiclass FSM and exo-command replay (with activity traces vs.\ ground truth where available). &
Twelve-class LSTM replaces the ECOC head; same activity names, FSM mapping, and replay figure structure as the multiclass SVM pipeline. \\
\end{tabular}
\end{center}

\begin{abstract}
\noindent
This document collects every pseudo-real-time pipeline PNG for the nine HuGaDB
\texttt{multi\_activity\_sequence} sessions used in \texttt{RunReplayGalleryBatch}.
Each session folder under \texttt{results/figures/pipeline/} holds four plots:
binary RBF SVM, binary LSTM, native twelve-class ECOC SVM, and native twelve-class LSTM,
with the same FSM assist gating (binary path) or activity-to-locomotion mapping (multiclass path).
Regenerate assets with \texttt{RunReplayGalleryBatch} from the MATLAB project root.
\end{abstract}
