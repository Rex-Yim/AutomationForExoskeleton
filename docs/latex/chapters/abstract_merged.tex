\subsection*{Main objectives}
This project aims to develop a software-centric enhancement module for a lower-limb robotic exoskeleton designed to assist workers in heavy load transportation tasks ($>20$~kg) in industrial, logistics, and construction environments. The primary objectives are to enable real-time detection of user locomotion intent (including walking, stair ascent/descent, sit-to-stand transitions, and speed variations), accurately estimate joint kinematics through multi-sensor fusion, and deliver adaptive personalized torque assistance to minimize physical effort and injury risk. To achieve this, the module will incorporate computational techniques to handle variability in user biomechanics and environmental conditions, ensuring robust performance across diverse scenarios.

\subsection*{Planned methods (interim report, Jan 2026 PDF)}
The proposed system integrates inertial measurement units (IMUs) placed on the lower limbs and back, with data processed through a Kalman filter-based sensor fusion pipeline implemented in MATLAB to achieve robust, low-latency joint angle and velocity estimation. Locomotion mode and transition prediction were planned using machine learning models (Support Vector Machines and Long Short-Term Memory networks) trained on labeled gait datasets, targeting $>90$\% classification accuracy and a 100--200~ms predictive horizon, with performance evaluated via confusion matrices, precision, and recall. Finite state machines were incorporated for locomotion mode transitions. The data collection protocol involves attaching three mobile phones as IMUs to the hips and lower back; performing standardized tasks such as walking 10~m on flat ground, climbing 10 steps, or navigating uneven surfaces at self-selected speeds; and exporting accelerometer, gyroscope, and magnetometer data as CSV files for processing. Joint kinematics information is intended for precise control.

\subsection*{Implemented methods (repository baseline, updated)}
The validated MATLAB baseline uses Kalman fusion (\texttt{imufilter}) as above. Classification uses overlapping windows (100 samples at 100~Hz, step 50).

The \textbf{primary deployment classifier} is a \textbf{binary} RBF SVM (\texttt{fitcsvm}, standardized features) with a \textbf{30-dimensional} training vector: five statistics per IMU slot $\times$ six slots (\texttt{Features.m}); HuGaDB supplies six on-body IMUs per window; USC-HAD and on-line simulation use one IMU stream with remaining slots zero-padded (\texttt{LocomotionFeatureVector}). The positive class groups \textbf{locomotion-like} activities (level walk, stairs, run per dataset label maps).

A \textbf{multiclass} ECOC ensemble (\texttt{fitcecoc}) provides twelve unified activity labels for analysis; exoskeleton gating uses the binary model and an asymmetric FSM (3 consecutive walk-like predictions to assist, 5 to release).

An \textbf{optional binary LSTM} (Deep Learning Toolbox) classifies the same locomotion-like vs.\ non-locomotion task from \textbf{sequence} inputs (raw IMU windows mapped to feature$\times$time matrices) via \texttt{TrainLstmBinary} and \texttt{RunExoskeletonPipelineLstm}; it is an alternative to the RBF SVM, not a hybrid SVM+LSTM fusion stage.

The 100--200~ms horizon remains a \textit{design target}; literature reports comparable latencies for kinematic lift-onset cues~\cite{chen2018}, but the submitted baselines were not benchmarked under that specific end-to-end metric.

\subsection*{Interim results statement (Dec 2025 / Jan 2026 PDF)}
The interim submission reported: a functional three-phone acquisition prototype; preliminary Kalman filtering and intention-detection algorithms validated on public datasets including HuGaDB and the USC-SIPI gait database; repository setup; real-time visualization; and initial locomotion classification with \textbf{98.88\%} accuracy and a \textbf{150~ms} predictive horizon. \textit{Corrections:} the smartphone gait database used in this project is \textbf{USC-HAD}~\cite{zhang2012}, not USC-SIPI. The 150~ms figure overlapped claims from the literature on hip-based lift detection~\cite{chen2018} and was not rigorously established as an end-to-end latency of the final classifier at submission time.

\subsection*{Updated empirical results (merged public data, repository metrics)}
Kalman filtering and SVM-based intention detection are implemented in MATLAB. Training uses \textbf{USC-HAD}~\cite{zhang2012} and \textbf{HuGaDB}~\cite{chereshnev2018} with merged windows and the 30-D feature layout. \textbf{Binary} five-fold out-of-fold accuracy from the committed evaluation files is \textbf{\BinaryOofAccUsc\%} (USC-HAD only), \textbf{\BinaryOofAccHuGa\%} (HuGaDB only), and \textbf{\BinaryOofAccMerged\%} (merged); the default saved binary model is merged. The twelve-class ECOC model achieves \textbf{\MulticlassOofAcc\%} OOF accuracy on a stratified subsample used for tractable cross-validation (see \texttt{results/multiclass\_evaluation\_metrics.mat}). Real-time pipeline simulation on CSV data shows fused hip flexion angle and FSM-latched assist commands. Scalar accuracies in this report are regenerated into \texttt{docs/latex/generated\_metrics.tex} by \texttt{scripts/ExportMetricsForReport.m}. Optional LSTM training yields a separate 20\% holdout metric (see Results; not comparable to five-fold SVM OOF) and may omit the confusion figure if training is interrupted before model export.

\subsection*{Main conclusions}
This project demonstrates that automation technology, particularly machine-learning-based intention recognition combined with sensor fusion and finite state machines, is applicable to next-generation powered exoskeleton control supervisors. When fully realized, the proposed module has strong potential for integration into industrial platforms (e.g., ExoTechHK Limited), offering high accuracy and balanced metrics on public IMU data while enhancing safety through hysteresis-based gating. This work contributes to workplace ergonomics and provides a reproducible foundation for hardware-in-the-loop studies. The repository now documents both the \textbf{original project plan} (hybrid SVM+LSTM fusion, extended fusion) and the \textbf{delivered MATLAB baseline} (binary + optional multiclass SVM, dual-dataset training, optional binary LSTM sequence classifier).
