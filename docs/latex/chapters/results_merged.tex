\section{Data Acquisition Prototype}
The project reached milestones aligned with the proposed methodologies during the prototype phase. A functional data acquisition prototype uses three mobile phones as IMUs (accelerometers, gyroscopes, magnetometers) to simulate placement on the lower limbs and back, enabling real-time collection. The protocol attaches phones to the hips and lower back; tasks include walking 10~m on flat ground, climbing 10 steps, and navigating uneven surfaces at self-selected speeds; data export to CSV ensures reproducibility. Preliminary simulations verified data integrity.

\textbf{Update:} Public-dataset training additionally relies on \texttt{usc\_had\_dataset.mat} and \texttt{hugadb\_dataset.mat} built by \texttt{LoadUSCHAD.m} and \texttt{LoadHuGaDB.m} when raw data are available.

\section{Sensor Fusion and Intention-Detection Algorithms}
\subsection*{Historical interim PDF (Jan 2026) vs.\ current repository}
The Jan 2026 interim PDF described a \textbf{five-dimensional} feature vector per window and quoted \textbf{98.88\%} five-fold accuracy on USC-HAD alone. The \textbf{current} codebase (Chapter~3) uses a \textbf{30-D} feature layout (six IMU slots $\times$ five features), merged USC-HAD and HuGaDB pools, and the accuracies in Table~\ref{tab:binarycv}---generated from \texttt{results/svm\_evaluation\_metrics\_*.mat} via \texttt{scripts/ExportMetricsForReport.m}. The Kalman pipeline and FSM remain: \textbf{asymmetric hysteresis} (3 consecutive walk-like predictions to assist, 5 to release).

Figure~\ref{fig:pipeline} (real-time pipeline output) illustrates the Kalman hip-angle trace and latched control command during walking, consistent with the FSM design.

\subsection*{Current quantitative results (merged datasets)}
The codebase trains on \textbf{merged} USC-HAD and HuGaDB windows using the 30-D standardized feature vector as described in Chapter~3. Table~\ref{tab:binarycv} lists five-fold \textbf{out-of-fold binary} accuracy for USC-HAD only, HuGaDB only, and merged training. Figure~\ref{fig:cm} shows the merged-model confusion matrix; Figure~\ref{fig:cm_mc} and \texttt{multiclass\_evaluation\_metrics.mat} document the twelve-class ECOC baseline.

\subsection*{Optional binary LSTM (sequence classifier)}
The repository includes an alternative \textbf{binary LSTM} trained on the same windowed USC-HAD/HuGaDB pools with raw IMU sequences (not the 30-D hand-crafted features). Training uses a stratified \textbf{20\% holdout} and a fixed random seed so metrics can be reproduced; this is \textit{not} directly comparable to the five-fold \textbf{out-of-fold} SVM accuracy in Table~\ref{tab:binarycv}. After training (\texttt{TrainLstmBinary}), run \texttt{EvaluateLstmConfusion} to export \texttt{lstm\_confusion\_matrix.png} and \texttt{lstm\_evaluation\_metrics.mat}. Re-run \texttt{ExportMetricsForReport} after evaluation so \texttt{generated\_metrics.tex} can include LSTM holdout metrics when that script is extended to read the LSTM \texttt{.mat} file.
