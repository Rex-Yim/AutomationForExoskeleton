\section{Summary of what was delivered}
This project delivers a reproducible MATLAB baseline for lower-limb exoskeleton assist gating on public IMU data. The final repository includes a HuGaDB-centered training and replay path, optional Kalman-based segment-pitch visualization, a 30-dimensional handcrafted feature pipeline, binary RBF SVM classification with five-fold out-of-fold evaluation, native multiclass ECOC baselines for per-dataset activity recognition, binary and multiclass LSTM sequence baselines, and finite-state-machine smoothing with asymmetric hysteresis for stable assist on/off behavior in simulation.

The strongest validated result is the HuGaDB streaming-policy binary SVM, which serves as the main deployment-oriented software baseline. USC-HAD remains useful as a benchmark comparison dataset under the same feature layout, while multiclass recognition and sequence models provide supporting analysis rather than the primary control claim. Metrics and figures are traceable to versioned scripts and saved outputs under \texttt{results/metrics/*/*.mat}, with report values exported through \texttt{ExportMetricsForReport.m}.

\section{Key limitations}
The submitted repository remains a software baseline rather than a full exoskeleton controller. Kalman fusion is used for visualization rather than mandatory classifier preprocessing. The implemented output is a stable binary assist command, not a torque-generation stack. Hardware integration, embedded deployment, controlled user trials, and latency benchmarking under real hardware conditions remain outside the validated scope of this report.

In addition, the binary SVM results use five-fold out-of-fold evaluation, whereas the LSTM baselines currently use separate stratified holdout splits. Those protocols are both useful, but they are not directly interchangeable, so the SVM path remains the clearest headline result. Multiclass accuracy is informative for analysis but is not yet strong enough to justify direct activity-dependent assistance scheduling.

\section{Future work}
\begin{itemize}
  \item Integrate the assist-gating supervisor with the exoskeleton hardware control stack and measure end-to-end latency under realistic load-transport tasks.
  \item Add embedded deployment and controller-side communication for real-time on-device execution.
  \item Extend validation with user studies, subject-specific calibration, and stronger personalization methods.
  \item Improve multiclass recognition and sequence-model evaluation with more stringent subject-wise validation and class-balancing strategies.
\end{itemize}
