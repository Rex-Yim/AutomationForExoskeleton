\section{Summary of what was delivered}
This project produced a reproducible MATLAB baseline for lower-limb exoskeleton \emph{assist gating} on public IMU data: a HuGaDB-centered replay path, optional Kalman visualization (\texttt{imufilter}), a 30-dimensional feature pipeline, binary RBF SVM classification with five-fold out-of-fold evaluation, optional native ECOC models (USC-HAD twelve classes; HuGaDB eight classes) for exploratory activity recognition, per-dataset binary LSTM sequence classifiers, per-dataset multiclass LSTM sequence baselines, and finite-state-machine smoothing with asymmetric hysteresis for stable on/off assist commands in simulation. Scalar accuracies and figures are traceable to versioned scripts and \texttt{results/metrics/*/*.mat}, with numbers exported into \texttt{generated\_metrics.tex} via \texttt{ExportMetricsForReport.m}.

The benchmark comparison table still shows that the binary accuracy target ($>90$\%) is met for USC-HAD training, while HuGaDB accuracy is lower, consistent with dataset heterogeneity and label mapping choices. The final repository trains on sixteen HuGaDB subjects and reserves subjects 06 and 08 for simulation. Multiclass accuracy remains modest and is not proposed as a deployable control layer without further work. End-to-end hardware trials, embedded deployment, hybrid SVM--LSTM fusion as in the original pseudocode, and the full predictive-horizon benchmark were not completed in software; the sections below preserve interim future-work bullets, timeline, risks, and repository status so planned scope stays distinguishable from the validated repository.

\section{Future work beyond the FYP (interim PDF)}
\begin{itemize}
  \item \textbf{Personalization:} Short calibration walks with transfer learning and lightweight RL (e.g., PPO/SAC) to tune assistance.
  \item \textbf{Injury-risk assessment:} Integrate ergonomic risk frameworks such as Zelik et al.~\cite{zelik2022} into user trials.
\end{itemize}

\section{Repository status (consolidated, 2026)}\label{sec:repo_status}
Relative to the interim plan, the following are \textbf{delivered in software}: HuGaDB-based binary RBF SVM training; dataset ablation scripts that preserve USC-HAD / HuGaDB comparison runs; optional Kalman visualization baseline; optional native ECOC SVMs per public dataset; binary and multiclass LSTM sequence classifiers per dataset (\texttt{TrainLstmBinary}, \texttt{EvaluateLstmConfusion}, \texttt{TrainLstmMulticlass}, \texttt{EvaluateLstmMulticlassConfusion}, \texttt{RunExoskeletonPipelineLstm}); asymmetric FSM; documentation and evaluation figures.

\textbf{Not delivered:} hybrid SVM+LSTM fusion as in the original pseudocode, MATLAB Coder deployment, ROS/PID torque layer, hardware controller integration, and prospective user trials---these remain aligned with the roadmap and risk tables above.
