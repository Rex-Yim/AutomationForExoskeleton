\noindent
\begin{minipage}[t]{0.318\textwidth}
  \posterbox{Why this project matters}{
    Workers carrying loads above 20~kg face high fatigue and musculoskeletal risk. For an exoskeleton to help, assistance must switch on when locomotion begins and switch off when the user stops, without invasive sensors or constant manual tuning.

    This project therefore focuses on a \textbf{software supervisor} rather than a full torque controller: detect locomotion-like intent from IMU data, estimate motion state, and issue a reliable assist gating signal.
  }

  \posterbox{Delivered pipeline}{
    \begin{compactitem}
      \item \textbf{Inputs:} public IMU datasets plus a three-phone CSV replay prototype.
      \item \textbf{Fusion:} Kalman-based hip-angle estimation using \texttt{imufilter}.
      \item \textbf{Windows:} 1.0~s at 100~Hz with 50\% overlap.
      \item \textbf{Features:} \textbf{30-D} vector, five statistics per IMU slot across six slots.
      \item \textbf{Classifier:} RBF SVM for binary locomotion-like vs.\ non-locomotion.
      \item \textbf{Supervisor logic:} FSM hysteresis, 3 consecutive positives to engage assist and 5 negatives to release.
    \end{compactitem}
  }

  \posterbox{Data and feature design}{
    HuGaDB contributes six on-body IMU streams per window. USC-HAD contributes one IMU stream, with the remaining slots zero-padded so both datasets share the same feature layout.

    That design makes the \textbf{merged} model more realistic than the USC-only score: it sees broader sensing layouts and more heterogeneous activity labels.
  }
\end{minipage}\hfill
\begin{minipage}[t]{0.318\textwidth}
  \posterbox{Validated quantitative result}{
    The strongest deployable claim is the \textbf{merged binary SVM}: it reaches \textbf{\BinaryOofAccMerged\%} five-fold out-of-fold accuracy while preserving a reasonable trade-off between false assist activation and missed locomotion onset.

    \vspace{0.14em}
    \setlength{\tabcolsep}{8pt}
    \renewcommand{\arraystretch}{1.14}
    \begin{tabular}{@{}>{\raggedright\arraybackslash}p{0.67\linewidth} r@{}}
      \toprule
      \rowcolor{accentsoft}
      \textbf{Training data} & \textbf{OOF acc.} \\
      \midrule
      USC-HAD only & \textbf{\BinaryOofAccUsc\%} \\
      HuGaDB only & \textbf{\BinaryOofAccHuGa\%} \\
      USC-HAD + HuGaDB & \textbf{\textcolor{accent}{\BinaryOofAccMerged\%}} \\
      \bottomrule
    \end{tabular}

    \vspace{0.16em}
    The USC-only score is numerically highest, but the merged model is the better poster result because it generalizes across a broader sensing setup.
  }

  \posterbox{How to read the other models}{
    \textbf{12-class ECOC (\MulticlassOofAcc\%):} useful as an exploratory activity-recognition baseline, but not yet strong enough for fine-grained control decisions.

    \par\smallskip
    \textbf{Binary LSTM:}
    \ifnum\HasLstmMetrics=1
      holdout accuracy is \textbf{\LstmHoldoutAcc\%}, but it was evaluated on a stratified 20\% split rather than five-fold OOF. It therefore remains a \textbf{separate baseline}, not evidence that it outperforms the SVM.
    \else
      metrics were not exported, so it is not part of the validated poster claim.
    \fi
  }

  \posterbox{Controller behavior}{
    The pipeline figure shows why the FSM matters: even with a high-accuracy classifier, raw predictions can chatter at transition boundaries. Hysteresis converts those predictions into a smoother and more usable assist signal for simulation-driven testing.
  }
\end{minipage}\hfill

  \posterbox{What is not claimed yet}{
    \setlength{\fboxsep}{6pt}%
    \fcolorbox{stripborder}{warnbg}{%
      \parbox{\dimexpr\linewidth-2\fboxsep-2\fboxrule\relax}{%
        \color{warntxt}
        \begin{compactitem}
          \item No hardware-in-the-loop or human-subject trials under load.
          \item The 100--200~ms predictive horizon remains a \textbf{design target}, not a re-benchmarked final result.
          \item Current output is \textbf{assist gating}, not full torque scheduling or terrain-aware control.
          \item LSTM and SVM have not yet been compared under the same validation protocol.
        \end{compactitem}
      }%
    }
  }

  \posterbox{Next steps}{
    \begin{compactitem}
      \item Align SVM and LSTM evaluation under the same cross-validation protocol.
      \item Add hardware integration, latency measurement, and load-carrying trials.
      \item Move from binary assist gating toward subject-specific, activity-aware assistance.
    \end{compactitem}
