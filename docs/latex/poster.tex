  \postersection{System overview}
  Sliding windows (1.0~s, 50\% overlap, 100~Hz): \textbf{30-D} feature vector (five statistics per IMU slot $\times$ six slots). HuGaDB uses six streams; USC-HAD single-stream windows are zero-padded.

  \par\smallskip\noindent
  \textbf{RBF SVM} (\texttt{fitcsvm}) is the primary classifier; \textbf{multiclass ECOC} (12 labels) supports analysis; \textbf{FSM}: 3 consecutive walk-like predictions to assist, 5 to release---reducing chatter in simulation.

  \postersection{Real-time pipeline (simulation)}
  Kalman-based hip angle, classifier output, and FSM-latched assist command---stable engagement vs.\ rapid toggling.

  \posterfig{pipeline_output.png}{Pipeline output: hip flexion, assist vs.\ acceleration magnitude, annotations when available}
\end{minipage}\hfill

\begin{minipage}[t]{0.31\textwidth}
  \postersection{Motivation}
  Heavy manual handling ($>20$~kg) drives MSDs, back pain, and lost productivity. Passive or fixed-profile exoskeletons often mismatch user intent and context.

  \par\medskip\noindent
  This work targets a \textbf{lightweight, software-based} module: real-time \textbf{locomotion vs.\ non-locomotion} gating using IMU data and classical ML---without EMG---for eventual hip/knee assistance during load transport.

  \postersection{Goals}
  \begin{itemize}\setlength{\itemsep}{0.18em}\setlength{\leftmargini}{1.2em}\setlength{\labelsep}{0.4em}
    \item Kalman-based fusion for joint kinematics from a \textbf{three-phone IMU} prototype (CSV replay).
    \item \textbf{Binary} locomotion intent (locomotion-like vs.\ non-locomotion) with SVM features on \textbf{USC-HAD} and \textbf{HuGaDB}.
    \item FSM smoothing for stable assist commands; optional \textbf{binary LSTM} sequence baseline.
    \item Metrics synced with \texttt{generated\_metrics.tex} / MATLAB exports.
  \end{itemize}

  \postersection{Takeaways}
  \begin{itemize}\setlength{\itemsep}{0.18em}\setlength{\leftmargini}{1.2em}\setlength{\labelsep}{0.4em}
    \item Reproducible \textbf{binary assist gating} on public IMU data with strong merged accuracy.
    \item \textbf{Kalman + SVM + FSM} integrated end-to-end in MATLAB for CSV-driven simulation.
    \item Next: hardware-in-the-loop, trials under load, aligned LSTM vs.\ SVM validation, end-to-end latency studies.
  \end{itemize}
