% ---------- Two columns ----------
\noindent
\begin{minipage}[t]{0.485\textwidth}
  \posterbox{Problem and approach}{
    \begin{compactitem}
      \item Manual load handling above 20~kg raises fatigue and musculoskeletal injury risk.
      \item An exoskeleton is only useful if assistance engages and releases at the right time.
      \item This work targets a \textbf{software supervisor}: infer locomotion-like intent from IMUs and issue a reliable assist on/off signal without EMG.
      \item The goal is not full torque control yet; it is a robust decision layer that can sit above future hardware control.
    \end{compactitem}
  }

  \posterbox{Implemented workflow}{
    \begin{compactitem}
      \item \textbf{Input:} public IMU datasets plus a three-phone CSV replay prototype.
      \item \textbf{Fusion:} Kalman-based hip-angle estimation using \texttt{imufilter}.
      \item \textbf{Windowing:} 1.0~s windows at 100~Hz with 50\% overlap.
      \item \textbf{Features:} \textbf{30-D} vector, five statistics per IMU slot across six slots.
      \item \textbf{Primary model:} RBF SVM for locomotion-like vs.\ non-locomotion.
      \item \textbf{Supervisory smoothing:} FSM hysteresis, 3 positives to engage assist and 5 negatives to release.
    \end{compactitem}
  }

  \posterbox{Data and evaluation protocol}{
    \begin{compactitem}
      \item \textbf{HuGaDB} contributes six on-body IMU streams per window.
      \item \textbf{USC-HAD} contributes one IMU stream; the remaining slots are zero-padded so both datasets share the same 30-D layout.
      \item The \textbf{binary SVM} is the deployment baseline and is evaluated with five-fold out-of-fold predictions.
      \item The \textbf{12-class ECOC} and \textbf{binary LSTM} are supporting baselines; the LSTM uses a separate stratified 20\% holdout and is therefore not directly comparable.
    \end{compactitem}
  }
\end{minipage}\hfill
\begin{minipage}[t]{0.485\textwidth}
  \posterbox{Main validated result}{
    The strongest deployable claim is the \textbf{merged binary SVM}: it reaches \textbf{\BinaryOofAccMerged\%} five-fold out-of-fold accuracy while preserving a practical trade-off between false assist activation and missed locomotion onset.

    \vspace{0.14em}
    \setlength{\tabcolsep}{8pt}
    \renewcommand{\arraystretch}{1.14}
    \begin{tabular}{@{}>{\raggedright\arraybackslash}p{0.67\linewidth} r@{}}
      \toprule
      \rowcolor{accentsoft}
      \textbf{Training data} & \textbf{OOF acc.} \\
      \midrule
      USC-HAD only & \textbf{\BinaryOofAccUsc\%} \\
      HuGaDB only & \textbf{\BinaryOofAccHuGa\%} \\
      USC-HAD + HuGaDB & \textbf{\textcolor{accent}{\BinaryOofAccMerged\%}} \\
      \bottomrule
    \end{tabular}

    \vspace{0.16em}
    The USC-only score is numerically highest, but the \textbf{merged} model is the better poster result because it generalizes across a broader sensing setup and label mixture.
  }

  \posterbox{How to read the supporting baselines}{
    \begin{compactitem}
      \item The pipeline plot shows why the FSM matters: it turns noisy frame-level predictions into a smoother and more usable assist signal.
      \item The multiclass ECOC score of \textbf{\MulticlassOofAcc\%} is informative for analysis, but not yet strong enough for fine-grained control.
      \item \ifnum\HasLstmMetrics=1 LSTM holdout accuracy is \textbf{\LstmHoldoutAcc\%}, but it uses a different validation protocol, so it is a supporting baseline rather than a direct SVM comparison.\else LSTM metrics are not exported, so it is not part of the validated poster claim.\fi
    \end{compactitem}
  }

  \posterbox{Current limits}{
    \begin{compactitem}
      \item No hardware-in-the-loop or human-subject load trials are reported here.
      \item The 100--200~ms predictive horizon remains a \textbf{design target}, not a re-benchmarked final result.
      \item Current output is \textbf{assist gating}, not full torque scheduling or terrain-aware control.
    \end{compactitem}
  }
\end{minipage}
