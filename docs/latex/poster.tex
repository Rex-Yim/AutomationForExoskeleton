\noindent
\begin{minipage}[t]{0.485\textwidth}
  \posterbox{Problem and approach}{
    \begin{compactitem}
      \item Manual load handling above 20~kg raises fatigue and musculoskeletal injury risk.
      \item An exoskeleton is useful when assistance engages and releases at the right time.
      \item This work targets a \textbf{software supervisor}: infer active movement from IMUs and issue a reliable assist on/off signal without EMG.
      \item The goal is not full torque control yet; it is a robust decision layer that can sit above future hardware control.
    \end{compactitem}
  }

  \posterbox{Implemented workflow}{
    \begin{compactitem}
      \item \textbf{Input:} canonical \textbf{HuGaDB} (GitHub v1 raw under \texttt{v1\_cleanup\_github/}) for training and held-out HuGaDB replay for simulation; USC-HAD is retained as a comparison benchmark.
      \item \textbf{Dataset handling:} loader applies the official gyro-corruption rules at read time, builds \texttt{hugadb\_dataset.mat}, and optionally reads \texttt{hugadb\_manifest.json} for per-session provenance and protocol metadata.
      \item \textbf{Fusion:} Kalman filtering is optional and used for a replay-time segment-pitch visualization trace, not for classifier preprocessing.
      \item \textbf{Windowing:} 1.0~s windows at 100~Hz with 50\% overlap.
      \item \textbf{Features:} \textbf{30-D} handcrafted vector, five statistics per IMU slot across six slots.
      \item \textbf{Primary model:} RBF SVM for \textbf{binary gating}: active-movement windows vs.\ inactive windows, using dataset-specific activity-to-binary maps.
      \item \textbf{Supervisory smoothing:} FSM hysteresis, 3 active predictions to engage assist and 5 inactive predictions to release.
    \end{compactitem}
  }
\end{minipage}\hfill
\begin{minipage}[t]{0.485\textwidth}
  \posterbox{Main validated result}{
    The strongest binary-SVM claim in the current repository is the \textbf{HuGaDB-trained} model: it reaches \textbf{\BinaryOofAccHuGa\%} five-fold out-of-fold accuracy on the HuGaDB benchmark split.

    \vspace{0.14em}

  }

  \posterbox{Data and evaluation protocol}{
    \begin{compactitem}
      \item \textbf{HuGaDB} contributes six on-body IMU streams per window, keeps the full native class coverage, and is the dataset used by the final pipeline.
      \item The canonical raw layout is \texttt{v1\_cleanup\_github/} (flat \texttt{HuGaDB\_v1\_*.txt}); the loader threads optional manifest metadata into the cache for validation and rebuild traceability.
      \item \textbf{USC-HAD} contributes one IMU stream; it remains in the poster to compare dataset effects under the same 30-D layout.
      \item The \textbf{binary SVM} is the main baseline; the active checkpoint is the HuGaDB model trained under the held-out-subject policy used by replay evaluation.
      \item \textbf{Binary} asks a simple control question: should assist stay \textbf{off} or turn \textbf{on}? \textbf{Multiclass} asks a harder recognition question: which exact activity is occurring?
      \item \textbf{Multiclass SVM} (12 USC / 12 HuGaDB classes) and \textbf{binary/multiclass LSTM} are supporting baselines; the SVM uses stratified subsamples and the LSTMs use separate stratified 20\% holdouts, so neither is a direct deployment comparison.
    \end{compactitem}
  }
\end{minipage}

\noindent
\begin{minipage}[t]{0.485\textwidth}
  \posterbox{How to read the supporting baselines}{
    \begin{compactitem}
      \item The pipeline plot shows why the FSM matters: it turns noisy window-level predictions into a smoother and more usable assist signal.
      \item The \textbf{binary} task is easier because several activities are merged into two groups; the \textbf{multiclass} task is harder because it must separate the exact activity label.
      \item Multiclass SVM OOF scores (\textbf{\MulticlassOofAccUsc\%} USC-HAD; \textbf{\MulticlassOofAccHuGa\%} HuGaDB) are informative for analysis, but not yet strong enough for fine-grained control.
      \item \ifnum\HasBinaryLstmMetrics=1 Binary LSTM holdout accuracy is \textbf{\LstmBinaryHoldoutAccUsc\%} on USC-HAD and \textbf{\LstmBinaryHoldoutAccHuGa\%} on HuGaDB; it uses a different validation protocol and remains a supporting baseline rather than a direct SVM comparison.\else Binary LSTM metrics are not exported, so they are not part of the validated poster claim.\fi
      \item \ifnum\HasMulticlassLstmMetrics=1 Multiclass LSTM holdout accuracy is \textbf{\LstmMulticlassHoldoutAccUsc\%} on USC-HAD and \textbf{\LstmMulticlassHoldoutAccHuGa\%} on HuGaDB; because the datasets remain distinct native label spaces, the comparison is a summary metric.\else Multiclass LSTM metrics are optional and are omitted until the per-dataset sequence models are trained.\fi
    \end{compactitem}
  }

  \posterbox{Current limits}{
    \begin{compactitem}
      \item No hardware-in-the-loop or human-subject load trials are reported here.
      \item The 100--200~ms predictive horizon remains a \textbf{design target}, not a re-benchmarked final result.
      \item Current output is \textbf{assist gating}, not full torque scheduling or terrain-aware control.
      \item Multiclass and LSTM baselines still need a stronger evaluation protocol aligned with the main binary SVM path.
    \end{compactitem}
  }
\end{minipage}\hfill
\begin{minipage}[t]{0.485\textwidth}
  \posterbox{Next steps}{
    \begin{compactitem}
      \item Align multiclass and LSTM evaluation with the same level of rigor used for the main binary SVM baseline.
      \item Add hardware integration, latency measurement, and load-carrying trials.
      \item Move from binary assist gating toward subject-specific, activity-aware assistance.
    \end{compactitem}
  }

  \posterbox{Summary}{
    \begin{compactitem}
      \item \textbf{Delivered:} canonical HuGaDB (GitHub v1) data flow, HuGaDB binary SVM, optional Kalman visualization, FSM smoothing, reproducible metrics, and replay in MATLAB.
      \item \textbf{Best validated result:} HuGaDB binary SVM achieves \textbf{\BinaryOofAccHuGa\%} five-fold OOF accuracy on the refreshed HuGaDB benchmark path.
      \item \textbf{System role:} simulation-level assist gating supervisor.
      \item \textbf{Value:} a credible, traceable software baseline that separates benchmark comparisons from the final deployed method.
    \end{compactitem}
  }
\end{minipage}
